Ser ikke ut som det finnes noe data. Har du satt opp ESP32 riktig? Venligst sjekk at ESP32 er flashed med riktig oppsett inne i .ino filen. Husk å bytt Wifi SSID og passord. Husk også at "Device name" kan ikke inneholde et mellomrom.

Under testing av flere ESP32 kontrollere støtte jeg på et problem hvor det kan hende de overskriver data. Dette betyr at en ESP32 sletter data til den andre siden API kallet kan gå litt tregt (Dweet sin feil). Det er ingen gode måter å gjøre dette på gratis, så hvis dette skjer bare prøv og start de opp på nytt igjen, eventuelt kan man sette opp delay i .ino koden. (Virket som 10 sekunder som den er på nå funker fint).

JS koden henter automatisk nye data på et 12 sekunders mellomrom, prøv å ikke refresh siden for mye siden man kan få en 'timeout' ved å sende for mange API kall. Hvis du får opp Ser ikke ut som det finnes noe data. erroren, kan dette være pga en timeout. Sjekk consollen.
